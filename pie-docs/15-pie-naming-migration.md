# Pie 네이밍 전환

## 목표

사용자 표시명을 Pie로 변경하면서 기존 설치, 설정, CLI, 딥링크, 원격 호스트, 모바일 페어링과
자동 업데이트가 갑자기 끊기지 않도록 단계적으로 전환한다.

기존 로컬 프로필의 계정 연결, 백업, 재실행, 복구 절차는
[데스크톱 배포와 수명주기](./16-desktop-lifecycle.md)를 함께 따른다.

## 즉시 적용 범위

- 새 제품 문서의 제품명은 `Pie`를 사용한다.
- 새 기능의 사용자 표시명과 화면 제목은 `Pie`를 사용한다.
- 새 도메인 용어는 `Pie Workspace`, `Pie Support`, `Pie Edge Agent`처럼 작성한다.
- 신규 파일과 모듈은 실제 도메인 이름을 사용하고 `orca-*` 접두사를 추가하지 않는다.

## 일괄 치환 금지 대상

다음 값은 기존 사용자 데이터나 외부 계약에 포함될 수 있으므로 단순 검색·치환하지 않는다.

- npm package 이름
- CLI 실행 파일과 명령어
- 앱 번들 식별자
- 앱 데이터 디렉터리
- 환경변수 접두사
- `orca://` 딥링크
- 페어링 URL과 토큰 payload
- SSH 원격 설치 경로와 Relay 소켓
- IPC 채널과 Runtime RPC 메서드
- 텔레메트리 이벤트와 저장된 설정 키
- 자동 업데이트 채널과 릴리스 파일명
- 모바일 앱의 저장된 페어링 정보

## 단계별 전환

### N1: 표시명

- 창 제목, 메뉴, About, 알림, 설치 화면을 Pie로 변경한다.
- 앱 아이콘과 브랜드 자산을 교체한다.
- 내부 패키지와 데이터 경로는 기존 값을 유지한다.
- 진단 화면에 표시명과 내부 런타임 식별자를 함께 보여준다.

### N2: CLI와 딥링크 별칭

- 신규 `pie` CLI를 제공한다.
- 기존 CLI는 같은 구현을 호출하는 호환 별칭으로 유지한다.
- `pie://` 프로토콜을 등록한다.
- 기존 딥링크도 일정 기간 수신하고 새 형식으로 정규화한다.
- 도움말과 새로 복사되는 링크는 Pie 형식을 우선한다.

### N3: 환경변수와 설정

- 신규 환경변수는 `PIE_` 접두사를 사용한다.
- 같은 의미의 기존 변수가 있으면 `PIE_`를 우선하고 기존 값을 fallback으로 읽는다.
- 두 값이 충돌하면 진단 로그에 출처를 기록한다.
- 설정 파일은 한 번만 마이그레이션하고 완료 버전을 저장한다.

### N4: 앱 데이터와 원격 Runtime

- 새 설치는 Pie 데이터 경로를 사용한다.
- 업그레이드는 기존 데이터 경로를 감지해 원자적으로 이전한다.
- 실패 시 기존 경로에서 계속 실행할 수 있어야 한다.
- SSH·WSL Relay는 구버전과 신버전 handshake를 구분한다.
- 구버전 원격 Runtime 연결을 지원하는 최소 호환 기간을 정한다.

### N5: 패키지와 배포 식별자

- 자동 업데이트가 기존 설치를 새 제품으로 정상 업그레이드할 수 있는지 플랫폼별 검증한다.
- 코드 서명, notarization, Windows installer upgrade code를 확인한다.
- 번들 식별자 변경이 OS 권한과 보안 저장소 접근에 미치는 영향을 검증한다.
- 모바일 페어링과 저장된 기기 토큰의 이전을 확인한다.

## 호환성 원칙

- 새 이름을 쓰기 위해 기존 사용자 설정을 버리지 않는다.
- 읽기는 구형과 신형을 지원하고 쓰기는 신형만 사용하는 기간을 둔다.
- 프로토콜 메시지의 브랜드 문자열보다 버전과 capability를 기준으로 호환성을 판단한다.
- 로그에는 비밀값 없이 어느 이름·경로·변수에서 값을 읽었는지 남긴다.
- 제거 시점은 사용량과 호환성 테스트를 근거로 결정한다.

## 코드 구조 원칙

- 사용자 표시명은 중앙 `ProductIdentity` 계약에서 공급한다.
- 앱 경로, CLI 이름, 프로토콜 scheme, 환경변수 이름을 표시명에서 파생하지 않는다.
- 기존 문자열을 새 코드에 복제하지 않고 호환 어댑터 한 곳에서 처리한다.
- 신규 코드 주석에는 단순 브랜드 변경이 아니라 호환 처리가 필요한 이유를 기록한다.

## 전환 전 확인

- 제품명과 로고의 상표·도메인 사용 가능성
- macOS·Windows 코드 서명 주체
- 설치 파일명과 자동 업데이트 URL
- CLI 패키지 배포 이름
- 앱 스토어와 모바일 앱 표시명
- 외부 문서, OAuth 앱, Webhook, API user-agent

## 완료 기준

기존 버전 사용자가 설정, 로그인, Workspace, SSH 세션, 모바일 페어링을 잃지 않고 Pie 표시명의
새 버전으로 업그레이드할 수 있고, 신규 설치는 Pie CLI·딥링크·데이터 경로를 사용해야 한다.
