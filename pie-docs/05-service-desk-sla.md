# 서비스 데스크와 SLA

## 목표

고객의 문의, 장애, 작업 요청을 계약·자산·프로젝트·원격 세션·코드 변경과 연결하고 응답과
해결 책임을 명확히 한다.

## 주요 화면

- 서비스 대시보드
- 티켓 대기열
- 내 티켓
- SLA 위험
- 주요 장애
- 유지보수 계약
- 정기점검 일정
- 고객 만족도

## 티켓 유형

- 문의
- 서비스 요청
- 장애
- 문제
- 변경 요청
- 결함
- 접근권한 요청
- 정기점검

## 티켓 기능

- 고객사, 담당자, 계약, 프로젝트 자동 연결
- 서비스와 자산 선택
- 제목, 설명, 첨부, 원문 채널
- 심각도, 우선순위, 영향 범위
- 담당팀과 담당자
- 고객 공개 답변과 내부 메모 분리
- 댓글, 멘션, 채팅, 회의
- 관련 티켓 병합·분리·연결
- 원격지원과 Workspace 시작
- 작업시간과 비용 기록
- 임시조치, 근본원인, 해결, 재발방지
- 고객 확인과 재오픈
- 만족도 조사

## 티켓 상태

`접수 → 분류 → 담당 배정 → 처리 중 → 고객 대기 → 해결 → 고객 확인 → 종료`

장애 대응 중에는 `에스컬레이션`, `주요 장애`, `완화됨` 상태를 사용할 수 있다. 고객 대기나
승인 대기 동안 SLA를 정지할 수 있는지는 계약 정책이 결정한다.

## SLA

- 고객사·계약·서비스별 정책
- 업무시간, 시간대, 공휴일
- 최초 응답, 작업 시작, 복구, 해결 목표
- 심각도별 목표
- 일시정지 조건
- 위반 임박과 위반 알림
- 담당자·팀·관리자 에스컬레이션
- 월별 준수율과 제외 사유
- 고객 보고서

## 주요 장애

- 장애 지휘자와 참여자
- 영향 서비스와 고객
- 상태 공지
- 실시간 활동 타임라인
- 채팅·회의·원격 세션
- 완화·복구 시점
- 사후 분석과 액션 아이템
- 문제 티켓과 재발방지 작업

## 유지보수 계약

- 지원 기간과 지원 시간
- 포함 공수와 차감 방식
- 초과 단가
- 무상 하자보수 기간
- 정기점검 주기
- 대상 서비스와 자산
- 연락·에스컬레이션 체계
- 월간 보고와 계약 갱신

## AI 지원

- 제목과 본문 요약
- 유형, 심각도, 담당팀 추천
- 유사 티켓과 지식문서 검색
- 반복 장애 감지
- 고객 답변 초안
- 원격 세션과 로그 요약
- RCA와 재발방지 초안
- 민감정보 자동 마스킹

## P1 완료 기준

고객이 앱에서 티켓을 만들고 진행상태를 확인하며, 엔지니어가 SLA 안에서 채팅·원격지원·코드
수정을 수행하고, 고객 확인과 작업시간 기록까지 하나의 티켓에서 완료할 수 있어야 한다.
