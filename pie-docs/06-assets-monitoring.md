# 자산과 모니터링

## 목표

고객의 서버, PC, 애플리케이션, 데이터베이스, 저장소, 배포환경을 식별하고 장애·변경·원격지원의
영향 범위를 설명할 수 있는 경량 CMDB를 제공한다.

## 자산 유형

- 데스크톱과 노트북
- 물리 서버와 VM
- 네트워크 장비
- 모바일·현장 장비
- 클라우드 계정과 리소스
- 애플리케이션과 서비스
- 데이터베이스와 메시지 시스템
- 저장소와 빌드 파이프라인
- 배포환경과 도메인
- 인증서와 라이선스

## 자산 정보

- 고객사, 사업장, 소유자, 관리자
- 호스트명, 운영체제, 아키텍처
- IP, 네트워크 구역, 시간대
- 설치 소프트웨어와 버전
- 보증, 구매, 교체, 계약 일정
- 보안 등급과 개인정보 처리 여부
- Edge Agent 상태와 마지막 접속
- 관련 프로젝트, 티켓, 변경, 원격 세션
- 구성 변경 이력

## 서비스 관계

- 상위 비즈니스 서비스
- 애플리케이션, API, DB, 인프라 의존성
- 저장소와 배포 파이프라인
- 담당팀과 공급자
- 장애 영향 고객과 사용자
- 변경 영향 경로

관계는 수동 편집과 수집 결과를 함께 사용한다. 자동 발견 결과는 검토 후 기준 정보로 승격한다.

## Edge Agent

- Pie 설치 패키지에 번들된 백그라운드 사이드카
- 고객 네트워크에서 중앙 Relay로 outbound 연결
- 장비 신원과 인증서 등록
- 상태·버전·기본 인벤토리 전송
- 승인된 원격 터미널·데스크톱 세션 호스팅
- 로그 수집과 Runbook 실행
- 자동 업데이트와 긴급 비활성화

## 모니터링

- 온라인 상태와 마지막 체크인
- CPU, 메모리, 디스크, 네트워크
- 프로세스와 서비스
- 포트, URL, 인증서 만료
- 로그 패턴과 이벤트
- 백업 결과
- 사용자 정의 점검 스크립트
- 임계치와 유지보수 시간대
- 경고에서 티켓 자동 생성
- 복구 시 티켓 갱신

## 원격 작업

- 서비스 시작·중지·재시작
- 로그 수집
- 프로세스와 포트 조회
- 시스템 정보 수집
- 승인된 스크립트 실행
- 재부팅과 재연결
- 파일 수집·배포

모든 원격 작업은 대상, 요청자, 승인자, 입력, 결과, 실행 시간을 기록한다.

## 권한 규칙

- 고객은 자사 자산만 조회한다.
- 비밀값은 일반 자산 필드에 저장하지 않는다.
- 자산 조회와 원격 조작 권한을 분리한다.
- 대량 작업은 별도 승인과 대상 미리보기를 요구한다.
- 에이전트가 오프라인이면 작업을 임의로 무기한 대기시키지 않는다.

## P3 완료 기준

고객 장비가 Edge Agent로 등록되고 상태와 기본 인벤토리를 전송하며, 경고가 관련 서비스와
자산이 지정된 티켓을 만들고, 승인된 원격 작업 결과가 감사 타임라인에 남아야 한다.
