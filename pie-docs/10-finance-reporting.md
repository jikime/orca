# 재무와 보고

## 목표

프로젝트와 유지보수에서 발생한 실제 공수·비용을 계약과 연결해 수익성을 파악하고, 고객과
내부 관리자가 신뢰할 수 있는 보고서를 만든다.

## 시간 기록

- 작업 타이머와 수동 입력
- 프로젝트, WBS, 티켓, 회의, 원격 세션 연결
- 업무 유형과 청구 가능 여부
- 정상, 야간, 휴일 시간 구분
- 설명과 증빙
- 제출, 검토, 승인, 반려
- 중복 시간과 비정상 입력 탐지
- 자동 수집 기록을 확정 전 후보로 제안

## 원가

- 직원별 또는 등급별 내부 원가
- 협력사 단가와 외주비
- 인프라, 라이선스, 구매, 출장 비용
- 프로젝트 공통비 배부
- 원가 단가의 적용 기간
- 과거 보고서 재현을 위한 스냅샷

## 청구

- 고정가
- 시간·재료 기반
- 마일스톤 기반
- 월 정액 유지보수
- 포함 공수 차감과 초과 공수
- 작업·티켓별 청구 후보
- 청구 제외와 조정 사유
- 청구 승인과 회계 시스템 전달
- 세금계산서·입금 상태 연동

Pie는 회계 원장을 직접 대체하지 않는다. 프로젝트·서비스 수행에서 발생한 청구 근거를 만들고
외부 회계·ERP 시스템으로 전달한다.

## 수익성

- 계약 금액과 변경 금액
- 계획·실제 매출
- 계획·실제 원가
- 프로젝트 매출총이익과 이익률
- 고객별 누적 수익성
- 티켓·서비스별 지원 원가
- 잔여 예산과 예상 완료원가
- 공수 소진 속도
- 손실 예상과 조기 경고

## 운영 보고서

- 프로젝트 진행률과 일정 편차
- 계획·실제 공수
- 인력 가동률과 과투입
- SLA 준수율
- 평균 응답·복구·해결 시간
- 장애 유형, 재발률, 재오픈률
- 원격지원 횟수와 시간
- 변경요청 발생과 승인 리드타임
- 테스트·결함·검수 현황
- 고객 만족도
- 지식 재사용과 자동화 절감시간

## 보고서 기능

- 역할별 대시보드
- 고객·프로젝트·기간·팀 필터
- 저장된 보기
- 차트에서 원본 데이터로 이동
- PDF, Excel, CSV 내보내기
- 정기 생성과 배포
- 고객 공개 필드와 내부 필드 분리
- 생성 시점과 데이터 기준일 표시
- 보고서 템플릿과 승인

## AI 분석

- 프로젝트 지연·적자 위험 요약
- SLA 위반 가능성
- 반복 장애와 비용 높은 고객 탐지
- 잔여 공수와 완료 시점 예측
- 경영 보고서 서술 초안
- 수치의 출처와 계산 기준 연결

AI가 생성한 해석은 원본 수치와 분리하고, 계산 결과 자체는 결정적 로직으로 산출한다.

## 권한 규칙

- 고객은 자사에 공개된 보고서만 조회한다.
- 내부 원가와 이익률은 재무 권한을 요구한다.
- 과거 단가 변경이 확정 보고서를 소급 변경하지 않도록 스냅샷을 사용한다.
- 대량 내보내기와 재무 데이터 접근을 감사한다.

## P3 완료 기준

승인된 시간 기록과 비용이 계약·프로젝트·티켓에 집계되고, 프로젝트 수익성과 고객용 SLA·작업
보고서를 동일한 기준 데이터에서 생성할 수 있어야 한다.
