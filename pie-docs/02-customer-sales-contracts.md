# 고객·영업·계약

## 목표

고객의 영업 정보와 실제 수행·지원 데이터를 연결한다. 범용 마케팅 CRM보다 SI 회사가 계약을
수주하고 프로젝트와 유지보수 서비스로 전환하는 흐름에 집중한다.

## 주요 화면

- 고객사 목록
- 고객 360도 상세
- 담당자와 조직도
- 영업 파이프라인
- 견적 편집기
- 계약 목록과 상세
- 갱신 예정 계약
- 고객 활동 타임라인

## 고객사 관리

- 법인명, 사업자 정보, 업종, 규모, 상태
- 본사, 지점, 사업장, 부서
- 업무 담당자, 의사결정자, 기술 담당자
- 담당 영업, PM, 지원팀
- 고객 등급, 태그, 보안 등급
- 연락처, 주소, 시간대, 업무시간
- 계약, 프로젝트, 티켓, 자산, 원격 세션 연결
- 고객별 매출, 공수, 수익성 요약
- 중요 활동과 최근 커뮤니케이션 타임라인

## 영업기회

- 리드와 영업기회 등록
- 예상 수주액, 확률, 예정일
- 제안요청서와 원문 첨부
- 경쟁사, 고객 예산, 의사결정 기준
- 활동, 다음 행동, 담당자
- 단계별 파이프라인과 정체 경고
- 프로젝트 유사사례와 기존 견적 검색
- AI 제안 범위와 리스크 초안

## 견적

- 구축, 개발, 라이선스, 인프라, 유지보수 항목
- 인력 등급별 단가와 예상 MM
- 고정가, 시간제, 월 정액, 마일스톤 청구
- 할인, 세금, 통화, 유효기간
- 내부 원가와 고객 표시 금액 분리
- 버전, 검토, 승인, 발송 이력
- PDF 내보내기와 전자서명 연동
- 승인된 견적에서 계약 초안 생성

## 계약

- 계약번호, 기간, 금액, 통화, 상태
- 본 계약과 변경 계약
- 구축 계약과 유지보수 계약 구분
- 과업 범위, 제외 범위, 납품물, 검수 조건
- SLA, 지원시간, 포함 공수, 초과 단가
- 청구 일정과 마일스톤
- 하자보수 기간과 유지보수 전환일
- 협력사와 하도급 계약 연결
- 갱신, 종료, 자동 알림
- 계약에서 프로젝트와 서비스 정책 생성

## 고객 360도

하나의 화면에서 다음 정보를 조회한다.

- 기본 정보와 담당자
- 진행·종료 프로젝트
- 활성 계약과 갱신 일정
- 열린 티켓과 SLA 위험
- 고객 자산과 시스템 상태
- 최근 채팅·회의·원격지원
- 누적 매출·원가·공수
- 고객 만족도와 반복 장애
- 미승인 변경요청과 검수 항목

## 권한 규칙

- 내부 원가, 이익률, 영업 메모는 고객에게 공개하지 않는다.
- 고객 담당자는 자사 정보만 조회한다.
- 계약 금액과 견적 할인은 별도 재무 권한을 요구한다.
- 고객 정보 내보내기는 감사 이벤트를 남긴다.

## 연계

- 프로젝트 수행
- 서비스 데스크와 SLA
- 자산과 모니터링
- 재무와 보고
- 이메일, 일정, 전자서명, 회계 시스템

## P1 완료 기준

고객사와 담당자를 등록하고, 계약에서 프로젝트와 SLA 정책을 생성하며, 고객 상세 화면에서
프로젝트·티켓·자산·지원 이력을 연결해 조회할 수 있어야 한다.
