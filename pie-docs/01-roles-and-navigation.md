# 역할과 내비게이션

## 목표

많은 기능을 한 앱에 넣되 모든 사용자가 모든 메뉴를 보게 하지 않는다. 역할, 현재 고객사,
프로젝트, 티켓 문맥에 따라 필요한 정보와 명령만 노출한다.

계정 등록, 로그인, 세션, permission 판정 규칙은
[회원가입·로그인과 RBAC](./01-authentication-rbac.md)를 따른다. 이 문서의 역할은 내비게이션과
업무 기본값을 정의하며 서버 인가를 대체하지 않는다.

## 전역 화면 구조

| 영역 | 책임 |
|---|---|
| 전역 레일 | 업무 포털, Workspace, 고객 관리, 지원, 대화와 관리 모듈 전환 |
| 문맥 사이드바 | 선택한 모듈의 Inbox, My Work, Project, 목록, 필터와 저장된 보기 |
| 주 작업 영역 | List·Board·Timeline, Project Overview, Workspace, 원격 세션 |
| 우측 인스펙터 | 담당자, 상태, Workspace·AgentSession·Artifact, 활동 타임라인 |
| 하단 패널 | 터미널, 실행 로그, 작업 큐, 진단 |
| 빠른 열기 | 고객, 프로젝트, WorkItem, AgentSession, 티켓, 파일, 명령, 원격 장비 통합 검색 |

전역 레일의 크기와 위치는 안정적으로 유지한다. 역할에 없는 항목은 빈 화면으로 남기지 않고
내비게이션에서 제거한다. 접근 가능한 URL이나 딥링크를 직접 열어도 권한 검사를 반복한다.

앱의 기본 시작 화면은 통계 dashboard가 아니라 사용자의 `My Work` 또는 지정한 SavedView다. Inbox는
알림·승인 요청, Intake는 외부·미분류 업무, Workspace는 terminal 실행을 다루며 서로 원본을 중복
소유하지 않는다. 상세 작업면은 [프로젝트 실행 모델](./27-project-execution-model.md), 실제 셸 배치와
상태 보존은 [데스크톱 UI 정보구조](./29-desktop-ui-information-architecture.md)를 따른다.

## 역할 정의

| 역할 | 기본 권한 |
|---|---|
| 조직 소유자 | 조직 소유권, 결제, 소유권 이전, 최고 관리자 지정 |
| 조직 관리자 | 조직 설정, 사용자, 정책, 전체 감사 |
| 사업 관리자 | 고객, 계약, 포트폴리오, 수익성 |
| 영업 | 고객, 담당자, 영업기회, 견적, 계약 초안 |
| PM | 프로젝트, 자원, 변경요청, 산출물, 검수 |
| PL | 프로젝트 작업, Workspace, 코드 검토, 배포 승인 요청 |
| 개발자 | 배정 프로젝트와 Workspace, Git, 테스트 |
| 지원 엔지니어 | 티켓, 고객 자산, 원격 세션, Runbook |
| 품질 담당자 | 요구사항, 테스트, 결함, 산출물, 검수 |
| 고객 관리자 | 자사 사용자, 요청, 승인, 검수, 보고서 |
| 고객 사용자 | 허용된 프로젝트와 티켓, 채팅, 원격지원 동의 |
| 협력사 | 명시적으로 배정된 프로젝트·업무·저장소 |
| 감사자 | 지정 기간의 읽기 전용 데이터와 감사 로그 |
| 게스트 | 단일 초대 세션과 제한된 채팅 |

## 권한 단위

- 조직
- 고객사
- 프로젝트
- WorkItem과 AgentSession
- 저장소와 Workspace
- 서비스 티켓
- 고객 자산과 장비
- 원격 세션
- 문서와 산출물
- 보고서와 재무 필드
- 자동화와 Runbook

권한은 역할 기본값과 리소스별 예외를 조합한다. 거부 규칙이 허용 규칙보다 우선한다.
기간이 있는 접근권한은 만료 시 자동 회수한다.

## 고객 모드

고객 모드는 기존 앱 셸을 사용하지만 다음 항목만 제공한다.

- 홈
- 참여 프로젝트
- 요구사항과 변경요청
- 티켓과 채팅
- 회의
- 산출물과 검수
- 원격지원 요청과 승인
- 자사 SLA·지원 보고서
- 고객 사용자 설정

고객 모드에서는 개발자 도구, 로컬 저장소 탐색, 셸 실행, 내부 원가, 내부 메모를 노출하지 않는다.

## 게스트 지원 모드

`pie://support/<invite>` 딥링크나 초대 코드로 진입한다. 로그인 없이 참여할 수 있는 경우에도
표시 이름, 초대 만료, 장비, 세션, 보기·조작 등급을 서버가 검증한다. 게스트는 초대된 세션
밖으로 이동할 수 없다.

## 공통 상호작용

- 모든 엔터티는 고유 URL과 복사 가능한 참조번호를 가진다.
- 목록 필터와 열 배치를 사용자별로 저장한다.
- 권한 인식 filter, group, sort와 layout은 SavedView로 저장·공유할 수 있다.
- 명령 팔레트는 현재 권한과 문맥에서 실행 가능한 명령만 보여준다.
- 알림에서 해당 고객·프로젝트·티켓·세션으로 바로 이동한다.
- 삭제보다 보관과 비활성화를 기본으로 사용한다.

## 완료 기준

동일한 설치본에서 내부 관리자, 개발자, 고객, 게스트로 로그인했을 때 서로 다른 내비게이션과
명령 집합이 표시되고, 숨겨진 IPC와 서버 API도 동일한 권한으로 차단되어야 한다.
