# 지식과 자동화

## 목표

해결된 티켓, 원격 세션, 프로젝트 문서, 배포 기록에서 재사용 가능한 지식을 만들고, 반복 작업을
승인 가능한 Runbook으로 전환한다.

## 지식 공간

- 조직 공통 지식
- 팀별 운영 지식
- 고객사 전용 지식
- 프로젝트 문서
- 제품·서비스별 지식
- 장애 해결 사례
- Runbook과 점검 절차
- 고객 공개 FAQ

## 문서 기능

- Markdown 기반 작성과 미리보기
- 이미지, 파일, 코드블록, 표
- 초안, 검토, 승인, 게시, 폐기 상태
- 버전과 변경 비교
- 소유자와 정기 검토일
- 고객 공개·내부·제한 등급
- 프로젝트, 티켓, 자산, 서비스, 변경과 연결
- 전문 검색과 의미 기반 검색
- 오래되거나 충돌하는 문서 탐지
- 문서 가져오기와 내보내기

## 지식 생성

- 해결된 티켓에서 해결 절차 초안
- 원격 세션에서 명령·결과 요약
- 주요 장애에서 RCA와 재발방지 문서
- 회의 전사에서 결정과 절차 추출
- 프로젝트 종료 시 인수인계 문서
- 반복되는 답변에서 FAQ 후보
- 코드와 설정 예시의 비밀값 제거

AI가 만든 문서는 출처를 연결하고 사람이 검토하기 전에는 공식 지식으로 게시하지 않는다.

| 구성 | 설명 |

| 트리거 | 일정, 티켓 상태, 경고, 승인, 수동 실행, Webhook |
| 조건 | 고객, 서비스, 심각도, 시간대, 자산 태그, 권한 |
| 작업 | 알림, 티켓 변경, 명령, 로그 수집, 배포, API 호출 |

| 보상 | 실패 시 중단, 되돌리기, 수동 전환 |
| 결과 | 출력, 상태, 증빙, 후속 작업, 감사 이벤트 |

## Runbook

- 버전이 있는 실행 정의
- 입력 파라미터와 유효성 검사
- 지원 운영체제와 실행 호스트
- 필요한 권한과 승인자
- 사전 점검과 예상 변경
- 단계별 명령과 타임아웃
- 성공 조건과 실패 처리
- 롤백 절차
- 민감 출력 마스킹
- 실행 이력과 결과 비교

## 대표 자동화

- 티켓 분류와 담당팀 배정
- SLA 임박 에스컬레이션
- 정기 로그·상태 수집
- 서비스 재시작
- 인증서 만료 알림
- 배포 전 점검과 배포 후 검증
- 고객 승인 후 변경 실행
- 주요 장애 채널과 회의 생성
- 해결 티켓의 지식문서 후보 생성
- 월간 유지보수 보고서 생성

## AI 실행 정책

- AI 제안과 실제 실행 권한을 분리한다.
- 고객·프로젝트별 허용 도구와 경로를 제한한다.
- `bypass` 성격의 권한은 일반 고객지원 세션에서 허용하지 않는다.
- 고위험 작업은 사람의 명시적 승인을 요구한다.
- 승인 요청에는 대상, 명령, 영향, 입력, 예상 결과를 표시한다.
- AI 입력과 출력의 보존·마스킹 정책을 조직별로 설정한다.
- 검색 문서, 티켓, 터미널 출력의 지시문은 신뢰할 수 없는 데이터로 취급한다.
- 모델이 제안한 도구 이름, 경로, URL, 명령을 정책과 스키마로 다시 검증한다.
- 고객·프로젝트별 허용 모델, 처리 지역, 외부 전송, 학습 사용 여부를 설정한다.
- 외부 모델의 기본값은 고객 데이터를 모델 학습에 사용하지 않는 계약으로 제한한다.
- 모델·도구별 호출량, 비용, 토큰, 동시 실행 quota를 둔다.
- 모델 장애와 rate limit 시 사람에게 전환하거나 승인된 대체 모델을 사용한다.
- 모델·prompt·도구 버전과 평가 결과를 고위험 자동화 릴리스에 연결한다.
- 대표 업무 세트로 정확성, 권한 우회, 비밀 유출, 위험 명령을 회귀 평가한다.

AI가 문장을 생성하는 기능과 실제 시스템을 변경하는 기능은 서로 다른 entitlement와 permission을
사용한다. 모델 출력은 권한 증명이나 승인 결과로 사용하지 않는다.

## P2 완료 기준

해결된 티켓과 원격 세션에서 출처가 연결된 지식문서 초안을 만들고, 승인된 Runbook을 지정
자산에서 실행해 결과와 감사 로그를 티켓에 첨부할 수 있어야 한다.
신뢰할 수 없는 입력이 도구 실행과 데이터 반출 정책을 변경하지 못하고, 모델·prompt 변경 전후의
핵심 평가 결과를 비교할 수 있어야 한다.
